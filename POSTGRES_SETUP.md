# Переход на PostgreSQL — Railway

## Зачем
JSON-файлы работают, но при 50+ клиентах и параллельных запросах возможны проблемы.
PostgreSQL на Railway — бесплатно до 1GB, надёжно, данные не теряются при передеплое.

## Шаги (займёт 10 минут)

### 1. Добавить PostgreSQL на Railway
1. Открой проект на [railway.app](https://railway.app)
2. Нажми **+ New** → **Database** → **Add PostgreSQL**
3. Railway автоматически создаст переменную `DATABASE_URL` в сервисе

### 2. Добавить переменные в Railway
В настройках сервиса (Variables) добавь:
```
STORAGE=sql
DATABASE_URL=<копируется автоматически из PostgreSQL сервиса>
RESET_DB=1   ← только при первом запуске, потом удали!
```

### 3. Передеплоить
Railway автоматически передеплоит после добавления переменных.

При первом старте с `RESET_DB=1` создадутся все таблицы.

> ⚠️ **Важно:** После первого успешного запуска удали `RESET_DB=1` из переменных — иначе база будет очищаться при каждом деплое!

### 4. Перенести существующие данные
Если в JSON-файлах уже есть заявки/няни — запусти скрипт миграции:
```bash
# Локально (подключись к Railway PostgreSQL через Railway CLI или connection string)
python3 migrate_json_to_sql.py
```

> Скрипт `migrate_json_to_sql.py` можно запросить у разработчика.

### 5. Проверить
- Открой `/healthz` — должен вернуть `{"ok": true}`
- Зайди в `/admin` — должны отображаться данные
- Отправь тестовую заявку с главной

## Что меняется
- Данные хранятся в PostgreSQL, а не JSON-файлах
- Работает функционал смен (`/api/nanny/me/shifts`, `/api/client/me/shifts`)
- Напоминания о сменах (`/cron/remind_2h`) становятся доступны
- Данные сохраняются между передеплоями (JSON-файлы иногда сбрасываются)

## Что НЕ меняется
- Все URL и API остаются прежними
- Интерфейс не меняется
- Telegram Mini App работает так же
