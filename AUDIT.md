## 1. БЕЗОПАСНОСТЬ 🔴 Критично

### 1.1 Отсутствует CSRF-защита
**Проблема:** Все POST-роуты (`/admin/nanny/save`, `/admin/shift/create`, `/admin/assign`, `/api/lead` и др.) не имеют CSRF-токенов.
Злоумышленник может форсировать действия от имени авторизованного пользователя через поддельную страницу.
**Решение:** Добавить `flask-wtf` с `CSRFProtect`, или реализовать `X-CSRFToken` заголовок для API.

### 1.2 FLASK_SECRET_KEY — дефолтное значение в коде
**Проблема:** `app.secret_key = os.environ.get('FLASK_SECRET_KEY', 'dev_secret_change_me')` — если ENV не выставлен, используется небезопасный ключ. На Railway это окружение — надо убедиться что переменная точно выставлена.
**Решение:** Убрать fallback, бросать `RuntimeError` если ключ не задан в production.

### 1.3 X-Frame-Options закомментирован
**Проблема:** `# resp.headers.setdefault('X-Frame-Options', 'SAMEORIGIN')` — защита от clickjacking отключена.
Telegram Mini App требует встраивания, но нужен более точный CSP `frame-ancestors` вместо полного отключения.
**Решение:** Добавить `Content-Security-Policy: frame-ancestors 'self' https://t.me https://*.telegram.org`

### 1.4 Отсутствует Content-Security-Policy (CSP)
**Проблема:** Нет CSP header вообще. Любой XSS может выполнить произвольный JS.
**Решение:** Добавить CSP с ограничением `script-src`, `style-src`.

### 1.5 dev_token проверяется только при FLASK_DEBUG=1 — OK
Это реализовано правильно, но нужно убедиться что на Railway `FLASK_DEBUG` не равен `1`.

### 1.6 Cron-эндпоинт `/cron/remind_2h` не защищён
**Проблема:** Любой может вызвать этот эндпоинт без авторизации, что отправит фейковые уведомления.
**Решение:** Добавить проверку секретного токена через заголовок `Authorization`.

### 1.7 Загрузка файлов — нет проверки размера расширения
**Проблема:** MIME-тип можно подделать в запросе. Расширение файла не валидируется.
**Решение:** Проверять расширение файла ИЛИ использовать `python-magic` для реальной проверки MIME по байтам.

## 2. ФУНКЦИОНАЛЬНОСТЬ 🟡 Средний приоритет

### 2.1 Режим SQL vs JSON — половина функционала не работает в JSON-режиме
**Проблема:** `api/nanny/me/shifts`, `api/client/me/shifts` возвращают `400: SQL mode required`. Telegram Mini App частично не работает без PostgreSQL.
**Решение:** Либо задокументировать это явно, либо реализовать JSON-fallback для основных API.

### 2.2 Уведомления Telegram падают молча
**Проблема:** В `_safe_send_message` ошибка логируется, но не всплывает. Если бот-токен неверный — заявки создаются, но никто не уведомляется.
**Решение:** Добавить индикатор в admin dashboard — «последнее успешное уведомление: XX минут назад».

### 2.3 Календарь — нет валидации прошедших дат на сервере
**Проблема:** На фронтенде прошедшие даты блокируются, но на API `/api/lead` можно отправить дату в прошлом.
**Решение:** Добавить серверную проверку `work_dates` и `meeting_date`.

### 2.4 Нет пагинации в /admin
**Проблема:** `load_leads()` грузит всё из JSON. При росте данных — медленно и OOM-риск.
**Решение:** Добавить пагинацию или лимит 100 последних.

### 2.5 Токены клиентов/нянь — предсказуемость
**Проблема:** `secrets.token_urlsafe(32)` — это нормально. ✅ Уже хорошо реализовано.

### 2.6 Cron `/cron/remind_2h` — нет Railway Cron настройки
**Проблема:** Эндпоинт есть, но нет документации как настроить его вызов на Railway (нет `railway.json` с cron).
**Решение:** Добавить `railway.json` с `"cronjobs"` или отдельный воркер.

## 3. SEO 🟡 Средний приоритет

### 3.1 sitemap.xml использует localhost в URL
**Проблема:** `<loc>http://localhost:5000/</loc>` — если запрос идёт с localhost (cron, healthcheck), Google получит неправильный URL.
**Решение:** Добавить env `SITE_URL=https://your-domain.railway.app` и использовать его в sitemap вместо `request.url_root`.

### 3.2 robots.txt — Sitemap URL тоже localhost
Та же проблема: `Sitemap: http://localhost:5000/sitemap.xml`

### 3.3 Страницы блога — нет canonical для пагинации
На `/blog` нет pagination, это нормально пока статей мало.

### 3.4 Schema.org — `ratingValue: "5"` захардкожено
**Проблема:** `"ratingValue": "5", "reviewCount": "{{ reviews|length }}"` — рейтинг всегда 5 независимо от реальных оценок.
**Решение:** Вычислять средний рейтинг из reviews.

### 3.5 OG Image — ссылается на logo-full.png
Логотип не оптимален как og:image (1200×630px рекомендовано). Нужна отдельная og-картинка.

### 3.6 Страница `/faq` — нет FAQ Schema
Добавить `"@type": "FAQPage"` schema для FAQ страницы — даёт rich snippets в поиске.

## 4. ДИЗАЙН И АДАПТИВНОСТЬ 🟢 Низкий приоритет

### 4.1 CSS файл — 1762 строки без минификации
На Railway статика раздаётся Flask-ом, без CDN/gzip. Нужно включить gzip через gunicorn или nginx.

### 4.2 Media queries только 4 breakpoints (1024/768/480px)
Нет breakpoint для 375px (iPhone SE), возможны проблемы на маленьких экранах.

### 4.3 Splash-экран — `splash.gif` — тяжёлый формат
Заменить на CSS-анимацию или Lottie для лучшей производительности.

### 4.4 Нет `loading="lazy"` на изображениях нянь
В шаблоне карточек нянь изображения грузятся без lazy-loading.

### 4.5 Нет skip-navigation link для accessibility
Добавить `<a href="#main" class="skip-link">`.

### 6.1 `tg_webapp.js` — нет fallback если авторизация упала
Если `/api/auth/telegram` вернул ошибку, пользователь видит пустой экран без объяснений.

`/client/app` не требует авторизации. Страница загружается, но данные без tg_id не придут.
